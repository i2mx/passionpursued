Tips



Learn by doing

When learning new things related to coding and programming, the sooner you go out and start messing around with the concepts that you are learning the faster you will learn them.



It's okay to ask for help

Some people might think that asking for help means that you don't understand something well enought. But thats not true, alot of programmers will still forget somethings sometimes and it's okay to look up the anwsers or ask for help. There are some sites such as Stack Overflow where you can ask for help from people who might be more experienced than you. So do it.



Comments

If you are working on a big project its a good idea to leave comments in your code so that when you come back to it you will be able to understand what everything does instead of being confused. In JavaScript it is // for the whole line or /* and */ from one point to the other in python it is # for one line and """ """ for from one point to another. Comments are diffrent in diffrent programming languges so it will depend on which on you are using.



Formatting

it is good to make your code readable for yourself. You should add indentations to your code to be able to tell what is where and what is what. When a line is getting too long you should try to go onto the next line at a convenient point. The capatalization of lines of text is also important and is called case convension, you can find out more about that here https://www.chaseadams.io/posts/most-common-programming-case-types/
